# Revenue Drivers & Customer Behavior Analysis

## Project Overview

This project analyzes transactional data from a UK-based online retailer to identify the key drivers of revenue and understand customer purchasing behavior.

The analysis focuses on:

* identifying revenue concentration among customer segments
* analyzing seasonal revenue patterns
* examining customer behavior using RFM (Recency, Frequency, Monetary) metrics
* understanding how different customer segments contribute to overall business performance

The goal is to demonstrate an end-to-end analytical workflow combining **Python (Pandas), SQL, and Power BI**.

---

# Dataset

Dataset: **Online Retail 2010–2011**

This dataset contains transactional data for an online retail store based in the United Kingdom.

Each row represents a **transaction line item**, including:

* Invoice number
* Product code
* Product description
* Quantity purchased
* Unit price
* Customer ID
* Transaction timestamp
* Country

---

# Data Cleaning (Python / Pandas)

Initial cleaning and feature engineering were performed in Python using Pandas.

Cleaning steps included:

* Removing rows with missing `CustomerID`
* Removing returned items (`Quantity < 0`)
* Removing invalid transactions (`UnitPrice <= 0`)

New analytical fields were created:

* **Revenue** = `Quantity × UnitPrice`
* **Year** extracted from `InvoiceDate`
* **Month** extracted from `InvoiceDate`

The cleaned dataset was saved as:

```
data/cleaned/online_retail_cleaned.csv
```

---

# SQL Data Model

The cleaned dataset was imported into SQL Server for structured analysis.

## Fact Table

### `online_retail`

This table contains the **transaction-level data**.

Each row represents a single line item purchased in an invoice.

Key fields include:

* `CustomerID`
* `InvoiceDate`
* `Quantity`
* `UnitPrice`
* `Revenue`
* `Year`
* `Month`

This table serves as the **fact table** in the analytical model.

---

## Customer Behavior Table

### `customer_rfm`

Customer-level behavioral metrics were calculated using SQL:

* **Recency** – days since the customer's most recent purchase
* **Frequency** – number of invoices per customer
* **Monetary** – total revenue generated by the customer

Each row represents **one customer**.

This table enables behavioral analysis of the customer base.

---

## Revenue Segmentation

### `customer_revenue_tier`

Customers were segmented based on cumulative revenue contribution.

A Pareto analysis was used to divide customers into three tiers:

| Tier   | Definition                                          |
| ------ | --------------------------------------------------- |
| Tier A | Customers generating the first 80% of total revenue |
| Tier B | Customers generating the next 15% of revenue        |
| Tier C | Customers generating the final 5% of revenue        |

This segmentation allows analysis of how different customer groups contribute to revenue.

---

# Power BI Data Model

The Power BI model follows a **star schema structure**.

```
DimDate
   |
online_retail
 |        |
customer_rfm
customer_revenue_tier
```

Tables used:

* `online_retail` – fact table
* `customer_rfm` – customer behavioral metrics
* `customer_revenue_tier` – revenue segmentation
* `DimDate` – calendar dimension

A dedicated **Date dimension** was created to support time-series analysis and Power BI time intelligence functions.

---

# Key Analysis Performed

The analysis included several SQL-based investigations:

### Revenue Concentration

A Pareto analysis was used to determine how revenue is distributed across customers.

### Revenue Contribution by Tier

Monthly revenue contributions from Tier A, B, and C customers were analyzed to determine which segments drive revenue.

### Nested Pareto Analysis

Revenue concentration within the Tier A segment was analyzed to determine whether the segment is dominated by a few "whale" customers.

### Customer Behavior by Tier

RFM metrics were compared across tiers to understand how purchasing behavior differs between customer segments.

---

# Key Insights

### 1. Revenue is highly concentrated among Tier A customers

Tier A customers represent approximately **26% of the customer base** but generate roughly **80% of total revenue**.

---

### 2. Seasonal revenue peaks are driven primarily by Tier A customers

Revenue peaks in **November 2011**, and the largest increases during peak months are driven by Tier A customers rather than lower-value segments.

---

### 3. Tier A is a broad base of high-value customers

Nested Pareto analysis showed that about **44% of Tier A customers generate 80% of Tier A revenue**, indicating that revenue within this group is relatively evenly distributed rather than dominated by a few extreme spenders.

---

### 4. Tier C customers show low engagement

Tier C customers contribute only a small portion of revenue and exhibit:

* high recency
* very low frequency
* low total spending

This suggests many are **one-time or inactive customers**.
